4. Admin/Manager Approval Workflow

Overview: Two-phase commit pattern where Admin makes changes and Manager approves/rejects them.
sequenceDiagram participant Admin participant AdminServlet participant FBS participant Database participant Manager participant ManagerServlet rect rgb(255, 240, 200) Note over Admin,Database: Admin Phase - Make Changes Admin->>AdminServlet: POST /ChangeFeatures.do
(New Prices, Seat Features) AdminServlet->>FBS: Get Features from Context Note over AdminServlet: Store Current Values in:
- newSeatPitch
- newSeatWidth
- newPrice
etc. AdminServlet->>FBS: Update Features (Temporary) FBS->>FBS: Set isChanged = true AdminServlet->>Admin: Redirect to ChangeFeatures.jsp
(Show Pending Changes) end rect rgb(200, 255, 200) Note over Manager,Database: Manager Phase - Review & Approve Manager->>ManagerServlet: GET /ApproveFeatures.jsp ManagerServlet->>FBS: Get Features with isChanged=true FBS->>ManagerServlet: Return Pending Changes ManagerServlet->>Manager: Display Changes for Review alt Manager Approves Manager->>ManagerServlet: POST /ApproveFeatures.do ManagerServlet->>FBS: Commit Changes FBS->>Database: UPDATE Features
(Make New Values Current) FBS->>FBS: Set isChanged = false Database->>FBS: Success ManagerServlet->>Manager: Redirect with Success Message else Manager Disapproves Manager->>ManagerServlet: POST /DisapproveFeatures.do ManagerServlet->>FBS: Revert Changes FBS->>FBS: Restore Old Values FBS->>FBS: Set isChanged = false ManagerServlet->>Manager: Redirect with Rejection Message end end
